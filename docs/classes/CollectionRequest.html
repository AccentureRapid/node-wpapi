CollectionRequest Class

Extends WPRequest
Defined in: lib/shared/collection-request.js:14
Module: CollectionRequest
Parent Module: WP

CollectionRequest extends WPRequest with properties & methods for filtering collections via query parameters. It is the base constructor for most top-level WP instance methods.

Example:

If auth credentials were already specified when the WP instance was created, calling .auth on the request chain will set that request to use the existing credentials:

request.auth().get...

Alternatively, a username & password can be explicitly passed into .auth:

request.auth( 'username', 'password' ).get...

filter

  • props
  • [value]
)
CollectionRequest chainable

Defined in lib/shared/collection-request.js:349

Specify key-value pairs by which to filter the API results (commonly used to retrieve only posts meeting certain criteria, such as posts within a particular category or by a particular author).

Parameters:

  • props String | Object

    A filter property name string, or object of name/value pairs

  • [value] String | Number | Array optional

    The value(s) corresponding to the provided filter property

Returns:

CollectionRequest:

The CollectionRequest instance (for chaining)

Example:

// Set a single property:
wp.filter( 'post_type', 'cpt_event' )...

// Set multiple properties at once:
wp.filter({
    post_status: 'publish',
    category_name: 'news'
})...

// Chain calls to .filter():
wp.filter( 'post_status', 'publish' ).filter( 'category_name', 'news' )...
