_renderQuery () String private

Defined in lib/shared/wp-request.js:321

Process the endpoint query's filter objects into a valid query string. Nested objects and Array properties are rendered with indexed array syntax.

Returns:

String:

A query string representing the specified filter parameters

Example:

_renderQuery({ p1: 'val1', p2: 'val2' });  // ?p1=val1&p2=val2
_renderQuery({ obj: { prop: 'val' } });    // ?obj[prop]=val
_renderQuery({ arr: [ 'val1', 'val2' ] }); // ?arr[0]=val1&arr[1]=val2

WPRequest chainable

Defined in lib/shared/wp-request.js:579

Set a requst to use authentication, and optionally provide auth credentials

Parameters:

  • [usrOrObj] String | Object optional

    A username string for basic authentication, or an object with 'username' and 'password' string properties

  • [password] String optional

    A user password string for basic authentication (ignored if usrOrObj is an object)

Returns:

WPRequest:

The WPRequest instance (for chaining)

Example:

If auth credentials were already specified when the WP instance was created, calling .auth on the request chain will set that request to use the existing credentials:

request.auth().get...

Alternatively, a username & password can be explicitly passed into .auth:

request.auth( 'username', 'password' ).get...
